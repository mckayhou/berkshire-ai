## 研究参考：AgentX (Kuaishou, arXiv:2606.26859)

论文标题：AgentX: Towards Agent-Driven Self-Iteration of Industrial Recommender Systems

### 与 berkshire-ai 架构映射

| AgentX 组件 | berkshire-ai 等价物 | 差距 / 可借鉴 |
|---|---|---|
| Brainstorm Agent（证据加权提案） | `research_loop.HypothesisProposer` + ExperienceRetriever | AgentX 从 4 类证据（历史实验/系统知识/数据分析/模型研究）检索；我们目前只有经验 few-shot + signal proposer |
| Developing Agent（代码实现 + dryrun） | `pipeline.run_full_cycle` D 段 + TextGrad 改写 | AgentX 有 DSL checker / feature switch / dryrun 三级安全网；我们有 validated_apply_gradient 验证门控 |
| Evaluation Agent（A/B → KEEP/EXTEND/DISCARD） | `realized_feedback` + `perf_metrics` | AgentX 接工业 A/B 平台；我们接 realized_price alpha |
| SGPO（semantic gradient → harness patch） | TextGrad `backward(scores)` + SkillForge bad-case patch | **最相关**：SGPO 对单个子 agent 做指令/规则修改，paired replay 防劣化 ≈ 我们的验证门控 |
| 失败根因写成可检索资产 | `ExperienceStore` + `FailureRecord` | AgentX 失败归因更结构化（维度: code_error/AB_regression/resource_conflict） |
| Paired replay 防劣化 | `validated_apply_gradient` (min_improvement) | 概念一致，但 AgentX 用旧轨迹 replay 做对比；我们用质量门控 |

### 关键启示（可纳入 V10.29+ ROADMAP）

1. **Evidence-weighted brainstorm**: 不仅从经验检索，还从系统知识库、数据异动、最新模型研究 4 路并行检索。→ 我们可扩展 `HypothesisProposer` 接入更多 evidence channel（如 aktools 异动扫描、研报摘要、graphify 知识节点）。

2. **Structured failure taxonomy**: AgentX 把失败分成 code_error / AB_regression / resource_conflict / idea_mismatch 并按类型路由不同修复动作。→ SkillForge 已有四维分析（context/logic/tool/compliance），可进一步对齐到 pipeline 级别而非仅 skill 级别。

3. **Paired trajectory replay for harness QA**: 修改 harness 后用旧成功轨迹 replay 验证不劣化。→ 我们 `trajectory_ab.py` + `run_ab_report` 已经在做离线 A/B；可进一步当 SkillForge patch 的自动回归门控。

4. **并发吞吐 vs 命中率 trade-off**: AgentX per-idea 命中率 2.7% < 人类 5.1%，但靠 8× 并发实验总产出更高。→ berkshire-ai 如果做多标的并行 R/D cycle，同理可以容忍较低 per-idea 精度但总体覆盖率更高。

5. **工具层依赖**: AgentX 强依赖快手内部平台（AB/feature store/DSL checker/dryrun/monitoring）。→ berkshire-ai 工具层相对薄（数据降级链 + aktools + perf_metrics）；想做真正的自动化闭环需要补齐：线上模拟环境 / paper-trade / 自动化验证 loop。

### 与现有参考文献关系

- RD-Agent (Microsoft): R/D 双循环范式 → 我们已有等价物（见 docs/rdagent_reference.md）
- AgentX: 把 RD-Agent 范式推到工业推荐系统的线上 A/B 闭环
- AEVO/AHE/CoEvolve: 通用 agent 进化 → 学术验证
- 我们: 投研决策领域的 TextGrad 闭环 + SkillForge bad-case 进化

### 结论

AgentX 是 berkshire-ai 路线在推荐系统领域的工业验证。核心思路（语义梯度 → harness 进化 → paired replay 防劣化）与我们已有架构高度同构。**不需要大改**，但可借鉴的方向：

- 多源 evidence brainstorm（V10.29 候选）
- trajectory replay 作为 SkillForge regression gate（V10.29 候选）
- 并发多标的 R/D cycle + 容忍低命中率高覆盖（V10.30+）
